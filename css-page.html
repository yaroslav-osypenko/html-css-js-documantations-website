Специфічність CSS важлива тема - особливо в великих проектах. це механізм, який використовує браузер для визначення, які стилі слід застосувати до елемента, якщо цей елемент має суперечливі правила CSS із різних селекторів. Це, по суті, система пріоритетів для CSS-правил. Чим вищою є специфічність селектора, тим більший пріоритет він матиме, і його стилі будуть застосовані, ігноруючи стилі з меншою специфічністю.

Специфічність обчислюється як чотиризначне число (A, B, C, D), де кожна позиція відповідає певному типу селектора. Жодна кількість селекторів нижчого рівня не може перевищити одиницю вищого рівня. Починається від A закінчується D. Також грає роль положення в самому CSS, якщо специфічність для елементів буде одинаковою, то свойство візьменться з нижчого селектора.

            Псевдоклас :not()
            Сам по собі :not() не додає специфічності, але він переймає специфічність селекторів,
            які знаходяться всередині нього.

            Псевдокласи :is() та :where()
            :is(): Бере специфічність найбільш специфічного селектора в його аргументах.
            Приклад: :is(#id-test, .class-test) матиме специфічність (0, 1, 0, 0)
            (специфічність #id-test).
            :where(): Завжди має нульову специфічність (0, 0, 0, 0).
            Це корисно для створення стилів, які легко перевизначити.

            4. Вплив !importantКлючове слово !important є спеціальним винятком із правил
            специфічності і його слід використовувати дуже рідко.!important ігнорує всі попередні
            розрахунки специфічності.Правило з !important матиме пріоритет над будь-яким іншим правилом
            (включаючи ID-селектори та інлайн-стилі), яке його не містить.
            Конфлікти з !important: Якщо два правила містять !important, перемагає те, що має вищу початкову специфічність.
            Якщо специфічність однакова, перемагає правило, оголошене пізніше у коді.
            Рекомендація: Уникайте використання !important, оскільки він ускладнює подальше налагодження та підтримку CSS-коду.
            Використовуйте його лише як крайній захід, наприклад, для тимчасових утилітарних класів.
            .

Звернення по вибірці значень з атрибуту. Є декілька варіантів використання і кожен працює по різному.

[href*="hh.ua"] - звертаємось до всіх обєктів, які мають в собі hh.ua, навіть якщо вони безпосередньо містять багато інших символів, якщо в їхній вибірці є hh.ua вони підпадуть під селектор.

[href^="https"] - Використовуючи вибірку ^= ми вибираємо елементи з атрибутом href які починаються на https

[href$=".ua"] - Вибираэмо елементи які закінчуються на .ua

[alt~="кіт"] - По аналогії з з виборкою *= де ми шукаємо певні не відокремлені символи в атрибутах Можна також шукати окремі слова в атрибутах. Під вибірку потрпалять атрибути в яких є відокремлене слово кіт

[class|="warning"] - Також можна вибрати тільки по слові, по якому починається клас, і далі іде дефіс, або слово одразу закінчується і містить тільки це слово.

[class*="cat" i] - Атрибут i в кінці буде відповідати за ігнорування регістру в вибірці.

Свойство display — це, мабуть, найважливіший інструмент у CSS. Воно визначає "правила гри" для елемента: як він виглядатиме, як займатиме простір і як взаємодіятиме із сусідніми елементами.

display: block (Блоковий)
Поведінка: Елемент займає всю доступну ширину (розтягується від краю до краю) і завжди починається з нового рядка.
Особливості: Можна задавати ширину (width) та висоту (height). Відступи (margin, padding) працюють з усіх боків.
Приклади за замовчуванням: div, p, h1, section.

display: inline (Рядковий)
Поведінка: Елемент займає лише стільки місця, скільки в ньому є контенту. Він не переноситься на новий рядок, а йде в рядок із текстом.
Особливості: Ширина та висота ігноруються. Вертикальні відступи (margin-top/bottom) зазвичай не працюють або не відштовхують сусідні елементи.
Приклади за замовчуванням: span, a, b.

display: inline-block (Рядково-блоковий) Поведінка: Гібрид. Елемент стоїть у рядку (як inline), але ви можете керувати його розмірами та відступами (як у block).
Використання: Часто використовується для кнопок, іконок або карток, що мають стояти поруч.
display: none - використовується для відключення елемента. Елемент перестає бути наявним, не відображається на макеті взагалі.

display: flex (Flexbox)
Суть: Одномірне макетування (контроль або по горизонталі, або по вертикалі).
Для чого: Ідеально підходить для вирівнювання елементів у рядок, центрування вмісту, розподілу простору між кнопками меню. Це зараз "золотий стандарт" для верстки компонентів. При заданні боксу display: flex, елемент стане блочним, а елементи всередині нього строчними.

display: inline-flex - При задані inline-flex, сам елемент якому задано стане строчним, і буде підлаштовуватись під ширину та висоту елементів всередині нього

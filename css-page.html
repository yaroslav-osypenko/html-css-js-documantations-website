Специфічність CSS важлива тема - особливо в великих проектах. це механізм, який використовує браузер для визначення, які стилі слід застосувати до елемента, якщо цей елемент має суперечливі правила CSS із різних селекторів. Це, по суті, система пріоритетів для CSS-правил. Чим вищою є специфічність селектора, тим більший пріоритет він матиме, і його стилі будуть застосовані, ігноруючи стилі з меншою специфічністю.

Специфічність обчислюється як чотиризначне число (A, B, C, D), де кожна позиція відповідає певному типу селектора. Жодна кількість селекторів нижчого рівня не може перевищити одиницю вищого рівня. Починається від A закінчується D. Також грає роль положення в самому CSS, якщо специфічність для елементів буде одинаковою, то свойство візьменться з нижчого селектора.

            D.  Елементи та Псевдоелементи
            Кількість селекторів елементів (тегів: p, h1, div)
            та псевдоелементів (::before, ::after).
            Приклад: div p::first-lineСпецифічність: (0, 0, 0, 2)
            .
            Особливі Випадки та Винятки.
            Універсальний селектор (*) та Комбінатори
            Універсальний селектор (*), комбінатори (>, +, ~)
            та псевдоклас :not() мають специфічність 0.
            Вони не впливають на підрахунок.
            Приклад: div > p має специфічність (0, 0, 0, 2)
            .
            Псевдоклас :not()
            Сам по собі :not() не додає специфічності, але він переймає специфічність селекторів,
            які знаходяться всередині нього.
            Приклад: div:not(.special) має специфічність (0, 0, 1, 1) (1 клас + 1 елемент).
            .
            Псевдокласи :is() та :where()
            :is(): Бере специфічність найбільш специфічного селектора в його аргументах.
            Приклад: :is(#id-test, .class-test) матиме специфічність (0, 1, 0, 0)
            (специфічність #id-test).
            :where(): Завжди має нульову специфічність (0, 0, 0, 0).
            Це корисно для створення стилів, які легко перевизначити.
            .
            4. Вплив !importantКлючове слово !important є спеціальним винятком із правил
            специфічності і його слід використовувати дуже рідко.!important ігнорує всі попередні
            розрахунки специфічності.Правило з !important матиме пріоритет над будь-яким іншим правилом
            (включаючи ID-селектори та інлайн-стилі), яке його не містить.
            Конфлікти з !important: Якщо два правила містять !important, перемагає те, що має вищу початкову специфічність.
            Якщо специфічність однакова, перемагає правило, оголошене пізніше у коді.
            Рекомендація: Уникайте використання !important, оскільки він ускладнює подальше налагодження та підтримку CSS-коду.
            Використовуйте його лише як крайній захід, наприклад, для тимчасових утилітарних класів.
            .
            5. Правило Тибрець (Tie-breaker)
            Якщо два селектори мають абсолютно однакову специфічність, застосовується правило послідовності:
            Перемагає правило, яке було оголошено пізніше в таблиці стилів.

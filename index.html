Тег meta — це самозакривний тег, який розміщується всередині head і використовується для надання метаданих (додаткової інформації) про HTML-документ. Ця інформація не відображається безпосередньо, але використовується браузерами, пошуковими системами та іншими вебсервісами. Тег meta не має закриваючого тегу.

name="viewport": Це повідомляє браузеру, що метадані стосуються області перегляду (Viewport) — видимої області сторінки.
Цей тег є критично важливим для адаптивного дизайну.

content="width=device-width, initial-scale=1.0": Це сама інструкція для області перегляду, що складається з двох частин:

width=device-width: Встановлює ширину області перегляду рівною фізичній ширині пристрою (замість використання ширини робочого столу за замовчуванням).
initial-scale=1.0: Встановлює початковий коефіцієнт масштабування сторінки на 100%. Це гарантує, що при першому завантаженні сторінка не буде масштабована браузером (наприклад, щоб вона виглядала меншою на мобільному телефоні).

Тег a — це парний рядковий елемент, що використовується для створення гіперпосилань, які дозволяють користувачам переходити на інші вебсторінки, файли, адреси електронної пошти або певні місця на тій самій сторінці. Основним атрибутом тегу є href, у якому вказується адреса призначення; без цього атрибута тег діє як звичайний текст. Окрім зовнішніх посилань, тег a часто використовується для створення "якорів" (навігації всередині документа) за допомогою ідентифікаторів id.. Якщо виникає питання що в контексті краще використати тег a чи button, відповідь проста Тег а, коли кнопка передбачає перехід на іншу сторінку, дзвінок, mail і навігацію по сторонці

Атрибут href (hypertext reference) — це найважливіша частина тегу a, яка вказує точну адресу, куди перейде користувач після натискання. Це може бути повний URL-шлях (наприклад, посилання на інший сайт), відносний шлях (посилання на іншу сторінку того самого сайту) або внутрішній ідентифікатор-якір, що починається з решітки (#), для швидкої навігації сторінкою. Якщо цей атрибут залишити порожнім, браузер сприйме посилання як перехід на поточну сторінку, оновивши її.

Атрибут href зі значеннями tel: та mailto: перетворює звичайне посилання на інтерактивну дію для зв'язку. Значення tel: (наприклад, tel:+380000000000) дозволяє ініціювати телефонний дзвінок безпосередньо з мобільного пристрою або через додатки для дзвінків на комп'ютері, тоді як mailto: (наприклад, mailto:example@mail.com) автоматично відкриває поштову програму за замовчуванням зі вже вписаною адресою отримувача. Не обовязково в тексті тега а вказувати номер, можна написати просто щось накшталт "подзвонити"

Атрибут target зі значенням _blank використовується в тезі a для того, щоб відкрити посилання в новій вкладці або новому вікні браузера. Це дозволяє користувачеві переглянути зовнішній ресурс, не закриваючи поточну сторінку вашого сайту. Для забезпечення безпеки та продуктивності при використанні цього атрибута рекомендується також додавати атрибут rel зі значенням noopener або noreferrer.

Атрибути noopener та noreferrer, які додаються до rel, є критично важливими для безпеки та продуктивності при використанні target="_blank". Коли ви відкриваєте нову сторінку, вона отримує доступ до початкового вікна через об'єкт window.opener, що дозволяє зловмисникам потенційно змінити адресу вашого сайту на підробну. noopener повністю розриває цей зв'язок, забороняючи новій сторінці керувати вашою, а також звільняє ресурси браузера, оскільки обидві вкладки працюють в окремих процесах. noreferrer виконує ті самі функції захисту, але додатково забороняє браузеру передавати адресу вашого сайту (HTTP-заголовок Referer) новому ресурсу, зберігаючи приватність ваших користувачів.

Тег button — це парний блочно-рядковий елемент, який використовується для створення клікабельних кнопок у вебпапежах. На відміну від тегу a, який призначений для переходів за посиланнями, button зазвичай ініціює дії на сторінці (наприклад, відкриття модального вікна, відправку форми або запуск скрипта). Усередині нього можна розміщувати не лише текст, а й інші HTML-елементи, такі як зображення або іконки, що робить його гнучким інструментом для створення інтерфейсів.

Тег img — це самозакривний (поодинокий) елемент, який використовується для вбудовування зображень у вебсторінку. Він є рядковим елементом за замовчуванням, але для нього можна задавати розміри прямо в коді або через стилі. На відміну від більшості тегів, він не містить тексту всередині, а відображає графічний контент, отриманий за вказаною адресою.

src (source): Обов'язковий атрибут, що вказує шлях до файлу зображення. там має бути файл формату .jpg, .png, .svg або .webp.
alt (alternative text): Важливий атрибут для доступності та SEO. Це текст, який з'явиться замість зображення, якщо файл не завантажиться, а також він зачитується скрінрідерами для людей з порушенням зору.
width та height: Визначають ширину та висоту зображення в пікселях. Рекомендується завжди вказувати їх, щоб браузер заздалегідь зарезервував місце під картинку, запобігаючи стрибкам контенту при завантаженні.

Тег video — це потужний парний елемент, призначений для вбудовування відеоконтенту безпосередньо у вебсторінку без сторонніх плеєрів. Він працює як гнучкий контейнер: ви можете вказати один файл через атрибут src або використовувати вкладені теги source, щоб запропонувати браузеру різні формати (наприклад, MP4 та WebM) для максимальної сумісності. Ключовим аспектом є атрибут controls, який додає стандартну панель керування (play, гучність, повний екран), без якої користувач не зможе взаємодіяти з відео, якщо тільки ви не створите власні кнопки через JavaScript.
Example:

Preload: Цей атрибут дозволяє вказати браузеру, як завантажувати відео під час завантаження сторінки. Значення none не вантажить нічого, metadata вантажить лише дані про тривалість та розмір, а auto намагається завантажити все відео одразу.Стилізація: На відміну від img, тег video часто потребує CSS-властивості object-fit: cover;,
щоб відео красиво заповнювало відведений йому блок без чорних смуг по боках.Текстові доріжки: Всередині video можна додавати тег track для підключення субтитрів (формат .vtt), що критично важливо для доступності контенту та SEO.
Автозапуск: Важливо пам'ятати, що більшість сучасних браузерів блокують autoplay, якщо відео не має атрибута muted, щоб не створювати раптового звукового шуму для користувача. Атрибут poster у тезі video дозволяє встановити зображення, яке буде відображатися як заставка (прев’ю) до того моменту, поки користувач не натисне кнопку відтворення або поки відео не почнеться автоматично; це критично для дизайну, щоб замість порожнього чорного вікна користувач бачив релевантну обкладинку. Атрибут playsinline є незамінним для мобільних пристроїв, особливо на iOS, оскільки він дозволяє відео відтворюватися безпосередньо в межах сторінки, не перемикаючи його автоматично в повноекранний системний плеєр, що зберігає цілісність вашого інтерфейсу.

Тег audio — це парний елемент, призначений для відтворення звукових файлів (музики, подкастів або голосових повідомлень) безпосередньо у вебдокументі. Як і у відео, атрибут controls є обов'язковим для відображення панелі керування (пуск, пауза, гучність), а використання вкладених тегів source з різними форматами та атрибутом type забезпечує сумісність із різними браузерами. Важливою практикою є додавання текстового повідомлення або посилання на завантаження всередині контейнера, яке спрацює лише в тому випадку, якщо браузер користувача застарілий і не підтримує цей тег.

Атрибути та особливості:
loop: Автоматично зациклює відтворення аудіо.muted: Запускає аудіо без звуку (часто використовується в комбінації з автозапуском).
preload: Керує тим, чи вантажити файл одразу (auto), лише метадані (metadata) чи не вантажити до натискання (none).
fallback (посилання): посилання з атрибутом download всередині тегу — чудовий спосіб забезпечити доступ до контенту для всіх користувачів.

Тег iframe (inline frame) — це парний елемент, який дозволяє вбудовувати сторонні вебсторінки, відео з YouTube, карти Google або інші віджети безпосередньо у ваш HTML-документ. Він створює своєрідне «вікно», всередині якого відображається контент з іншого URL-джерела, вказаного в атрибуті src. Це незамінний інструмент для інтеграції сторонніх сервісів, проте він потребує уваги до безпеки та продуктивності, оскільки завантаження чужих сайтів може сповільнити сторінку.

Ключові атрибути та налаштування: src: Адреса контенту, який потрібно відобразити (наприклад, посилання на відео або мапу).
width та height: Визначають розміри фрейма. Рекомендується задавати їх через CSS для кращої адаптивності.
frameborder: Застарілий атрибут (в HTML5 краще використовувати CSS-властивість border: none;), який визначає наявність рамки навколо вікна.
allowfullscreen: Дозволяє контенту всередині iframe (наприклад, відеоплеєру) переходити в повноекранний режим.
loading="lazy": Сучасний атрибут, який змушує браузер завантажувати iframe лише тоді, коли користувач наближається до нього при прокручуванні, що значно пришвидшує перше завантаження сайту.
sandbox: Важливий атрибут для безпеки, який накладає обмеження на контент у iframe (наприклад, забороняє виконання скриптів або надсилання форм), якщо вони не дозволені явно.
